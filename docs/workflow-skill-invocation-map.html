Skill invocation map

From nothing to an idea to a finished piece of content

This is the current operator path through InfluencerOS. The conductor skill sequences the work; each stage loads its own instruction file, context row, schema, and CLI gate where applicable.

Workflow overview Format and script decisions Generation OS workflow Skill registry Context matrix
Invocation rule

From the repo root, load skills/<skill>/SKILL.md. From a Creator Workspace, load .claude/skills/<skill>/SKILL.md plus any sibling SKILL.local.md.

High-Level Chain

The first two phases create the creator and discover opportunities. The middle phases select and plan one content unit. The final phases package the artifact and optionally register performance evidence.

00

Create creator foundation

create-influencer runs setup subskills and validates readiness.

01

Research current patterns

influencer-os routes to create-research-findings.

02

Create ideas

manage-idea-queue writes scored queue entries.

03

Approve one idea

promote-idea creates the human-approved promotion and project.

04

Plan the piece

apply-social-template then create-production-plan.

05

Package final content

Generate or import assets if needed, review quality, then create-output-package.

After Finished Content

These steps are not required to make a finished content package. They close the performance loop after a human publishes and metrics exist.

Publication

register-published-post

Records a post a human already published or scheduled. It never posts by itself.

Analytics

ingest-analytics

Adds manual or CSV Analytics Snapshot records against live Published Post Records.

Interpretation

create-performance-summary

Maps performance evidence to packaging, hook, body retention, payoff, and CTA.

Learning

distill-creator-learning

Writes evidence-linked creator lessons into memory so future research and ideas improve.

Memory write

memory-write

Optionally promotes a durable fact into context/MEMORY.md with the size cap enforced.

Session close

wrap-up

Captures process learnings and reconciles skill or context documentation after deliverables.

Instruction Sources Loaded Around The Skills

These files tell the agent what context to read and which local rules override the base skill.

Conductor

skills/influencer-os/SKILL.md

Owns sequencing, provenance, halt rules, provider boundary behavior, and routing to producer skills.

Context loading

docs/os-construction/context-matrix.md

Names which creator and OS files to load for each workflow row: setup, research, planning, packaging, analytics, and learning.

Skill registry

docs/os-construction/skill-registry.md

Lists triggers, reads, writes, and override support for every repo-owned skill.

Product language

CONTEXT.md

Defines Creator, Project, Social Post Format, Format Subtype, Output Package, gates, reviews, passes, and warnings.

Runtime override

SKILL.local.md

Optional creator-specific or OS-specific corrections. Local rules override base skill rules for that scope only.

Validation

schemas/ plus CLI

Schemas define record shape. CLI write gates and validate project enforce cross-record references.

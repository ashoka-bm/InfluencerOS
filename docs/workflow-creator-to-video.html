InfluencerOS workflow map

Creator setup to created video

A file-first view of the path from creating an influencer foundation to producing a generated video asset. Each step shows the durable records, schemas, skills, templates, and CLI seams that make the step auditable.

Workflow overview Skill invocation map Generation OS Pipeline contract
Scope

Stops when the project has a video file under generation/assets/ and a matching generation/asset-manifest.json row. Packaging, publishing, analytics, and learning are downstream.

Run setup from intake, guided interview, or generated draft. Output is a Creator Workspace that can reach content_ready or generation_ready.

Step 03

Research current opportunities

Create dated evidence and concise findings. If real videos were inspected, create a Video Understanding Pack before synthesizing findings.

The human approves the exact idea, formats, platforms, evidence, schedule claim, creative carry-forward, and Projects to create.

Step 06

Create Project shell

Promotion writes the locked Idea Promotion, creates projects/<project-slug>/, and replaces the scaffold with a production-facing evidence brief.

workspace-library/creators/<slug>/research/idea-promotions/*.json workspace-library/creators/<slug>/projects/<project>/project.json workspace-library/creators/<slug>/projects/<project>/evidence-brief.md python3 -m influencer_os init-project
Step 07

Apply social template

Choose the structure that best carries the promoted idea through hook, retain, payoff, CTA, and optional packaging beats.

skills/apply-social-template/SKILL.md schemas/applied-social-template.schema.json docs/social-template-library.md projects/<project>/plan/applied-template.json
Step 08

Draft short-form video plan

For format_short_form_video, write the Micro-Journey Video Plan and provider-neutral Base Video Generation Plan.

skills/create-production-plan/SKILL.md schemas/micro-journey-video-plan.schema.json schemas/base-video-generation-plan.schema.json projects/<project>/plan/generation-plan.json
Step 09

Approve exact generation call

The human approves a specific provider, model, plan ref, prompt refs, output filenames, scope, batch cap, and cost note.

skills/request-generation-approval/SKILL.md schemas/generation-approval-record.schema.json docs/provider-boundary.md python3 -m influencer_os record-generation-approval
Step 10

Create or import video asset

Generation dispatch consumes the approval once, or an already-created video is imported with origin, tool, license, and warnings recorded.

influencer_os/providers/dispatch.py skills/import-generated-asset/SKILL.md schemas/generation-asset-manifest.schema.json projects/<project>/generation/assets/final-video.*
Step 11

Record generated state

The created video is auditable when the file exists, the manifest row points to it, and validation resolves the approval or import provenance.

influencer_os/generation.py examples/generation-asset-manifest.example.json projects/<project>/generation/asset-manifest.json python3 -m influencer_os validate project

Review before packaging

After the video exists, generated or imported media needs a passing QualityReview before it can become upload-ready material.

Boundaries

Where the workflow stops

Human gates

Two explicit approvals

The idea promotion gate approves production work. The generation gate approves the exact provider call or bounded batch. Neither approval can be inferred from prior intent, keys, or plan text.

Created video

Definition of done

A video has been created when the artifact exists under generation/assets/, a manifest row binds it to approval or import provenance, and validate project resolves the chain.

After creation

Not yet upload-ready

Packaging is downstream. Generated and imported media must pass review-generated-assets before create-output-package can register upload-ready files.

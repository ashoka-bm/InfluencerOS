Decision workflow

Where format, subtype, outline, script, and prompts are decided

InfluencerOS separates the content container from the craft subtype and from the actual production material. Broad format is approved before project creation. Subtype and script-like detail live in production planning.

Workflow overview Skill invocation map Generation OS workflow Format shortlist
Important gap

The current system has no first-class script record. Video scripts are implicit in the micro-journey plan, shots, and prompts. Article and thread copy starts in plan fields and becomes upload-ready package material later.

Subtype Rules

Subtype is optional. It names the craft when the container alone is too broad.

Article

format_subtype

Allowed values: essay, reported_feature, newsletter_dispatch.

An op-ed currently has no dedicated enum. It should be treated as essay or omitted until the schema adds op_ed.

Carousel

format_subtype

Allowed values: designed_slides, photo_set.

The subtype decides whether the plan emphasizes graphic slide design or photo selection and sequencing.

Thread

format_subtype

Allowed values: chain, single_post.

The subtype decides whether the text expands across ordered posts or compresses into one throughline post.

Video

No subtype enum yet

Talking head, demo, POV, skit, explainer, challenge, and similar video types are currently represented by template choice, beat mapping, shot design, and generation prompts.

Image

No subtype enum yet

Craft is expressed through central visual idea, composition, avatar or scene requirements, overlay policy, and prompt.

Story

No subtype enum yet

Craft is expressed through frame roles, sequence arc, lightweight text or sticker notes, and continuity requirements.

What Counts As The Script Today

The current pipeline plans content before it writes a final asset. It does not yet store a standalone script file with its own schema.

Short-form video

Implicit script

hook, retain.setup, retain.escalation, payoff, cta_or_loop, and shots[] define the beat script.

Generation prompt

Provider-neutral prompt plan

prompt_sequence[] in the Base Video Generation Plan turns shots into provider-neutral prompts. It is still not approval to generate.

Article

Outline before final copy

working_title, deck, thesis, and sections[] define the article. Final text belongs in upload-ready package files.

Thread

Draft copy in the plan

opening_post and posts[].copy carry the thread draft directly in the production plan.

Recommended future contract

Make scripts first-class when the workflow needs stricter authorship

A future slice should add a dedicated script or draft artifact if operators need versioned voiceover scripts, article bodies, dialogue, narrated scenes, or platform-specific final copy before packaging. That would make the current implicit script layer explicit and reviewable.

# Commercial Expression Is Typed And Pressure Is Derived

Every Campaign Concept declares one primary Commercial Function and optional supporting Functions; each Project selects one execution Function from that approved set. Concept Approval sets maximum Offer Integration (`absent`, `embedded`, `contextual`, `central`) and CTA Intensity (`none`, `soft`, `direct`), while Project stores exact planned values at or below those ceilings. Commercial Pressure is derived OS-wide as `none`, `low`, `moderate`, or `high`, never assigned manually.

The fixed matrix is: absent plus none is `none`; absent plus soft is `low`; embedded plus none or soft is `low`; contextual plus none is `low`, plus soft is `moderate`, and plus direct is `high`; central plus none is `moderate`, and plus soft or direct is `high`. Absent or embedded offer integration with a direct transactional CTA is invalid. Pressure Indicator v1 maps the four tiers to 0, 1, 2, and 3 and reports the normalized weighted mean on a 0-100 scale while retaining tier counts.

The initial 3:1 guidance is advisory: the current calendar horizon reports each platform's high-pressure share and warns above 25%, but never blocks Concept Approval or Project creation. Cross-creator benchmarks, experiments, evidence maturity, adaptive policy, mandatory rolling windows, and automated harm stops remain deferred.

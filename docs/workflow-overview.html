InfluencerOS workflow

From creator context to learning-backed future ideas

InfluencerOS is a typed local pipeline. Each step writes a durable record, validates the next handoff, and keeps human approval at the risky boundaries.

Current state

Planning, creative review, Generation OS, packaging, publication registration, analytics ingestion, performance summary, and learning distillation are built. Real paid generation adapters are not installed by default.

Main Pipeline

The chain starts with creator-specific context and ends with creator-specific learning. Automation, publishing, and provider calls stay behind explicit gates.

01 Context

Creator profile and schedule

Load the creator identity, audience, niche, content strategy, readiness, reference library, and schedule state.

02 Evidence

Research and video understanding

Capture dated evidence. When real videos shape the idea, store timestamp-aware Video Understanding Packs.

03 Backlog

Idea Queue

Add scored ideas with hook, premise, payoff, creator fit, evidence refs, schedule fit, and format recommendations.

04 Human gate

Idea Promotion

The user approves the promotion package. Approved platforms and formats become the production surface.

05 Project

Project creation

A project locks one content unit type, source refs, acceptance criteria, reference assets, and status.

06 Structure

Applied Social Template

Choose the content structure that pulls the audience from hook to retain to payoff to CTA.

07 Plan

Format-specific production plan

Route to article, thread, carousel, story, image, or micro-journey video schema.

08 Advisory

Creative review and passes

Review hook and payoff. Run clarity or voice passes when useful. These produce advice, not hard stops.

09 Generation OS

Approval, assets, and quality

Record exact generation approval, dispatch or import assets, write provenance, and run the blocking QualityReview.

10 Package and learn

Output, publication, analytics, memory

Register upload-ready assets, record human publication, ingest analytics, summarize performance, and distill lessons.

Controls

InfluencerOS separates gates, reviews, passes, and warnings so agents know what can block the pipeline.

Human gate

Idea Promotion

Blocks project creation until the user approves the promotion package.

Human gate

Generation approval

Blocks image, video, audio, render, upload, paid, or irreversible provider calls until the exact call or bounded batch is approved.

Blocking review

QualityReview

Blocks packaging for generation-sourced media unless identity, continuity, technical, and creator-boundary checks pass.

Advisory review

Hook/Payoff Review

Flags creative issues. It may recommend revision or block, but it does not stop packaging by itself.

Advisory pass

Writing and voice passes

Rewrite for clarity or creator voice. The author decides what to keep.

Warning

Project warnings

Surface concerns such as platform fit or stale evidence without changing the project status.

Traceability invariant

Every final artifact should resolve upstream

Packaging validation proves the chain from upload-ready material back to the project, production plan, promotion, research evidence, creator profile, and generation approval when media was generated or imported.

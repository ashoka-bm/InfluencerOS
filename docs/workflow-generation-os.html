Phase 3 workflow

Generation OS: exact approval, provenance, and blocking quality review

Phase 3 adds the provider-safety spine between production planning and output packaging. It does not install a paid provider by default; the mock adapter is the only built-in adapter.

Boundary rule

Key presence is never approval. Every generation call or bounded batch needs a GenerationApprovalRecord that matches the exact provider, model, plan refs, prompt refs, and requested assets.

Phase 3 Flow

The media path can start with dispatch or import. Both paths must land in the same manifest and pass the same packaging gate.

Input

Generation-ready plan

The project has a production plan. Short-form video also has plan/generation-plan.json.

Human gate

Record exact approval

request-generation-approval packages the exact call or bounded batch. The approval statement is captured verbatim.

Asset entry

Dispatch or import

Dispatch consumes an approved record. Import brings in existing user-provided or externally generated media.

Ledger

Write asset manifest

Every file in generation/assets/ gets a row with origin, file hash, artifact path, and provenance.

Blocking review

QualityReview

Generated and imported media must pass identity, continuity, technical, and creator-boundary checks.

Package gate

Register Output Package

Upload-ready media must cite a manifest row and a passing QualityReview before packaging succeeds.

Files Phase 3 Adds

All Phase 3 state stays inside the Project's generation/ subtree, except reference-library approvals for reusable reference assets.

Approval

generation/approval-records/

One JSON file per approval. Status moves through draft, approved, executing, executed, or cancelled.

Artifacts

generation/assets/

Flat directory for generated, imported, or user-provided media files. Subdirectories and symlink escapes fail validation.

Provenance

generation/asset-manifest.json

The ledger binds assets to approval records, provider call metadata, plan prompt refs, import source, license, warnings, and file hashes.

Review

generation/quality-reviews/

QualityReview records name the manifest rows they cover. Packaging requires a passing review for each generation-sourced media asset.

Safety Rules

These are validator-enforced where possible, not just operator notes.

Exact approval

No approval, no call

dispatch_generation requires an approved record id. It refuses missing, draft, cancelled, executing, or already executed records.

Single use

Approvals cannot be reused

Dispatch flips approved to executing before adapter work and then to executed. Regeneration needs a new record.

Hard stop

Kill switch wins

INFLUENCER_OS_DISABLE_PAID_CONNECTORS disables generation dispatch even when an approval record exists.

Import honesty

Unknown rights stay unknown

Imported assets record source, tool, license, attribution, and warnings. Unknown license becomes a warning, not a guessed right.

Manifest parity

Files and rows reconcile

An orphan file, dangling manifest row, tampered hash, unresolved approval, or wrong artifact path fails project validation.

Blocking gate

Media cannot skip review

Generated and imported media need a passing QualityReview before they can appear as upload-ready media. Text roles are exempt.

Packaging provenance

What validation proves for a packaged media asset

The Output Package must resolve the media asset through the generation ledger before it can ship.

# InfluencerOS Operator

This file stores stable operator preferences for work on InfluencerOS itself.

Keep it short. Do not store secrets, personal private data, or creator-specific facts here.

## Preferences

- Build from the Agentic OS reference instead of reinventing from scratch.
- Surface any divergence from Agentic OS before implementation.
- Use deterministic inputs, outputs, acceptance criteria, and validation.
- Prefer local-first behavior until remote access is explicitly approved.
- Use test-driven, context-driven, and domain-driven development.
- Keep construction docs separate from runtime workflow contracts.
- Use visual maps for architecture and workflow decisions when they reduce ambiguity.
